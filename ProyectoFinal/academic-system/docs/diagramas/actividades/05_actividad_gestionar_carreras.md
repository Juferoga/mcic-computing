## 2. Actividades del Flujo

| ID | Actividad | DescripciГіn |
|----|-----------|-------------|
| A1 | Seleccionar opciГіn | El usuario hace clic en "Gestionar Carreras" |
| A2 | Mostrar panel | Se carga la interfaz con la lista de carreras |
| A3 | Seleccionar acciГіn | El usuario elige entre agregar, consultar, modificar o eliminar |
| A4 | Mostrar formulario | Se presenta el formulario para ingreso de datos |
| A5 | Ingresar datos | El usuario llena los campos del formulario |
| A6 | Validar datos | Se verifican las reglas de negocio (ID Гәnico, campos obligatorios) |
| A7 | Guardar en BD | Se persiste la informaciГіn en PostgreSQL |
| A8 | Buscar en BD | Se consulta la carrera por ID |
| A9 | Mostrar resultado | Se exhiben los datos encontrados o mensaje de error |
| A10 | Confirmar eliminaciГіn | Se pide confirmaciГіn antes de eliminar |
| A11 | Eliminar de BD | Se ejecuta el DELETE en la base de datos |

---

## 3. Reglas de Negocio Aplicadas

1. **ID Гәnico**: No pueden existir dos carreras con el mismo ID
2. **Nombre obligatorio**: El campo nombre no puede estar vacГӯo
3. **Integridad referencial**: No se puede eliminar una carrera si tiene estudiantes asociados
4. **Registro de calificaciГіn**: Campo opcional pero Гәnico en su formato

---

## 4. Excepciones

| CГіdigo | DescripciГіn | AcciГіn |
|--------|-------------|--------|
| E01 | ID de carrera ya existe | Mostrar mensaje de error y solicitar nuevo ID |
| E02 | Nombre de carrera vacГӯo | Resaltar campo y mostrar mensaje de validaciГіn |
| E03 | Error de conexiГіn a BD | Mostrar mensaje de error y ofrecer reintento |
| E04 | EliminaciГіn restringida | Mostrar mensaje indicando estudiantes asociados |
